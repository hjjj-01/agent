## 修复 LangChain 导入错误（云服务器部署）
- **问题**：`wms_agent.py` 第23行导入了 `from langchain_core.agents import create_agent`，该函数仅存在于 LangChain 1.0+，但 requirements.txt 指定 `langchain>=0.3.0`，云服务器安装了 0.3.x 版本导致 ImportError
- **根因**：`create_agent` 被导入但从未使用，实际代码用的是 `langchain.agents` 的 `create_tool_calling_agent`（0.3.x 可用）
- **修复**：删除了无用的 `create_agent` 导入行
- **教训**：AI 生成的代码可能包含未使用的导入，本地能跑是因为本地碰巧装了更高版本的库；建议部署时用 `pip freeze` 固定版本号

## 知识库改造成 ID 映射表（第三次同步）

**背景**：用户将 WMS 模块的 WMS 数据改为只实现员工工作量查询，并将知识库内容改为存储员工ID和岗位ID，用于和 KPI 请求/响应进行匹配。

**新设计**：
- 知识库（id_register）：员工姓名→worker_id、岗位名→node_id 的映射表
- WMS API 参数：nodeIdList（岗位ID列表）、endTime（日期范围）
- 流程：用户说"张三的工作量"→工具先查知识库找 worker_id → 调 WMS API → 过滤该员工数据

**修改的文件：**
1. `src/agent/tools.py`
   - `WMSTools.__init__` 增加 `rag_system` 参数（用于查知识库找ID）
   - 新增 `_lookup_id_from_knowledge()` 方法：通过 RAG 检索从知识库中提取 worker_id 和 node_id
   - `query_employee_work()` 增加 `worker_name` 和 `node_name` 参数
   - 工具 description 完整重写，说明知识库 ID 映射机制
2. `src/agent/wms_agent.py`
   - 系统提示词更新：说明知识库用于 ID 映射查找
3. `src/rag/document_processor.py`
   - `create_sample_documents()` docstring 更新为 ID 映射说明
   - 添加顶部注释块说明知识库结构和使用方式

   - logger.success 消息更新
